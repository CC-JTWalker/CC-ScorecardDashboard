# Governance

The repository owner is the final maintainer and release manager. Changes are proposed through pull requests, discussed in the pull request, and merged only after required checks and review pass.

Material security decisions should record the threat, alternatives considered, chosen mitigation, and residual risk. Release tags should be signed and produced only from reviewed commits on `main`.

Additional maintainers may be added after sustained, constructive contributions. Code ownership and release permissions should be reviewed when maintainers change roles.
